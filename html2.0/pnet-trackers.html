DotGNU Portable.Net Bugs and Patches

The bug tracker is here and the patch tracker is here.

Guidelines for submitting bugs and patches

  • Please be logged into Savannah when submitting a bug report or patch.
  • Please check the current open bugs to make sure that your bug hasn't been reported already.
  • Please test with the current cvs version to make sure that your bug hasn't been fixed already.
  • Bug reports should contain sufficient information to reproduce the bug.
  • Please isolate the part of your program or library which doesn't work properly because of the bug to a small test case, and attach that to your bug report.
  • If the bug is easy to fix, please submit a patch which fixes it instead of a bug report.
