DEE - Distributed Execution Environment

The plan for DEE is that it will be an extension of SEE which allows to logically join several instances of DGEE, the DotGNU Execution Environment into a single powerful and reliable webservice execution engine. The component DGEE instances will generally run on several, geographically distributed, computers.

In this respect DEE is similar to VRS. The difference is that DEE is a centrally administered system while VRS is a dynamic peer-to-peer system.

DEE is currently still "dreamware", implementation work has not started yet. If you're interested, you're welcome to start the development.
