DotGNU Portable.NET

The goal of this project is to build a suite of free software tools to build and execute .NET applications, including a C# compiler, assembler, disassembler, and runtime engine. While the initial target platform was GNU/Linux, it is also known to run under MS Windows, Solaris, NetBSD, FreeBSD, OpenBSD and MacOS X. The runtime engine has been tested on the x86, PowerPC, ARM, Sparc, PARISC, s309, Alpha, and IA-64 processors.

Focus on Compatibility

DotGNU Portable.NET is focused on compatibility with the ECMA specifications for CLI. There are other projects under the DotGNU meta-project to build other necessary pieces of infrastructure, and to explore non-CLI approaches to virtual machine implementation.

Frequently Asked Questions

Please refer to the FAQ of the Portable.Net development project.

Download

Released versions are available from the downloads page. It may however be best to grab the latest development version from Savannah CVS. This project's CVS repository can be checked out through anonymous (pserver) CVS with the following instruction set. When prompted for a password for anoncvs, simply press the Enter key. 

  cvs -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/dotgnu-pnet login

  cvs -z3 -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/dotgnu-pnet co .
NOTE: Before you can build the cvs version, you need to run auto_gen.sh in order to create the Makefile.in and configure files.

Building

The three primary components of the system must be built in the following order: treecc, pnet, and then pnetlib.

You will need flex and bison to build DotGNU Portable.NET. We use flex 2.5.4 and bison 1.28. Some support is provided for the BSD version of yacc, but bison is recommended. Other versions of lex or yacc may not work.

Each source package can be unpacked, built, and installed in the usual manner:

$ zcat <package-version.tar.gz | tar xvf -
$ cd package-version
$ ./configure
$ make
$ make install

Running the examples

Once you have built the system, you can test it on the examples within the "samples" directory:

$ cd samples
$ ../engine/ilrun hello.exe
$ ../engine/ilrun fib.exe

The PNetMark program, listed above, is a more involved example. Instructions for running PNetMark can be found within its README file.

Latest Changes

26 July 2003:

Version 0.5.10 of pnet, pnetlib, pnetC, and ml-pnet have been released. A lot has happened this release. The System.Windows.Forms namespace is coming along well, thanks to the help of many people in the DotGNU community - particularly Simon Guindon, Neil Cawse, Richard Baumann, Ian Fung, Cecilio Pardo, and the relentless Gopal V. The C compiler is now working enough to allow people to write real C applications. We need assistance on the pnetC library, to port glibc across to run in the Portable.NET environment. Thong Nguyen has been working very hard on threading this month, and has achieved a lot! Threading is now working quite well - help with testing on various platforms would be appreciated. Richard Baumann has also been working on System.Reflection.Emit, and is making good progress. The CVM unroller was also rewritten into a generic framework that should be much easier to port to new architectures. For more information, see the NEWS files.

17 July 2003:

Version 0.2.6 of treecc has been released. Minor feature additions that don't affect Portable.NET, but may affect other users.

17 June 2003:

Version 0.5.8 of pnet, pnetlib, pnetC, and ml-pnet have been released. A lot of things have happened this cycle. The Java and Visual Basic language front-ends for cscc have been added to the system, and a huge number of new classes have been implemented in pnetlib, including support for IPv6 and IrDA networking. We have also begun implementation of the System.Windows.Forms library, built on top of the X11 library. This means that it will be able to run on any X11-capable system. Currently tested to work on x86-based GNU/Linux systems and ppc-based MacOS X systems. We are looking for volunteers to help us implement the "x11-winforms" library as quickly as possible, and to test it on as many platforms as possible. Finally, the ml-pnet project has been started to provide ports of certain Mono C# libraries. System.Data (ADO.NET) currently works, and we hope to have System.Web (ASP.NET) working soon.

3 May 2003:

Version 0.5.6 of pnet, pnetlib, and pnetC have been released. The two big changes this release are the signature-compatibility of pnetlib (described below), and improved support for stripped-down profiles for embedded systems.

26 April 2003:

The CVS version of pnetlib's mscorlib.dll is now 100% signature-compatible with the .NET Framework SDK. Read the full announcement on the DotGNU developer's mailing list here.

1 April 2003:

A small patch for version 0.5.4 pnet has been released. It applies to all platforms except MacOSX, and fixes a problem when trying to compile pnetC. If you don't need to use the C compiler, this patch should not affect you.

29 March 2003:

Version 0.5.4 of pnet, pnetlib, and pnetC, and version 0.2.4 of treecc have been released.

12 February 2003:

Version 0.0.6 of pnetmark has been released. This version adds the SciMark and Linpack floating-point benchmarks, plus better support for building and running with other CLI implementations.

8 February 2003:

Version 0.5.2 of pnet and pnetlib, and version 0.2.2 of treecc have been released. This is mostly a cleanup and portability release. Portable.NET is now known to run on i386, PowerPC, ia64, alpha, PA-RISC, sparc, arm, and s390 systems.

16 January 2003:

I wrote a simple Mahjongg tile game for Portable.NET and Qt# 0.6. You can download the sources for the game here. It's very simple at the moment. Feel free to send me patches to improve the gameplay, add loadable tilesets, scoring, or any other cool feature that you can think of. Adam Treat was kind enough to provide a screenshot.

11 January 2003:

Version 0.5.0 of pnet, pnetlib, and pnetC have been released. This is the most stable version of the compiler, runtime, and libraries to date. Version 0.2.0 of treecc, version 0.0.4 of pnetmark, and version 0.0.2 of pnetcurses have also been updated.

23 November 2002:

Version 0.4.8 of pnet, version 0.4.8 of pnetlib, and version 0.1.8 of treecc have been released. The version numbers of pnet and pnetlib have been synchronized because they are always released together. The biggest news for this release is that cscc can now cleanly compile pnetlib, and the pre-built binaries in pnet/samples are produced with cscc. We are now independent of third party C# tools, and it should be a lot easier to help out on the C# library. Please help us if you can by writing C# classes or test cases.

22 November 2002:

The FAQ has been updated with a lot more information on installing and using Portable.NET, the C compiler, and how to contribute.

5 November 2002:

Version 0.4.6 of pnet, version 0.3.0 of pnetlib, and version 0.1.6 of treecc have been released. It is now possible to compile pnetlib using pnet, but that is no guarantee that it will run just yet. All help will be appreciated in helping track down the remaining compiler bugs.

2 September 2002:

Version 0.4.4 of pnet, version 0.2.8 of pnetlib, version 0.0.2 of pnetC, and version 0.1.4 of treecc have been released. The biggest change to this release is the introduction of a C compiler front-end for cscc. When combined with the pnetC C library, this can be used to build C applications to run on CLR platforms such as pnet as pure IL bytecode.

25 July 2002:

Version 0.4.2 of pnet and version 0.2.6 of pnetlib, have been released.

17 July 2002:

There have been some DNS problems with the dotgnu.org domain recently. The dotgnu.info domain can be used as a back up until we can resolve these issues.

Version 0.4.0 of pnet, version 0.2.4 of pnetlib, version 0.0.3 of pnetmark, and version 0.1.2 of treecc have been released.

12 June 2002:

We have uploaded a description of how Portable.NET can be used in embedded environments here. The process of making Portable.NET embedded-compatible is still ongoing. Check the CVS version for the latest information.

10 June 2002:

Version 0.3.8 of pnet and version 0.2.2 of pnetlib, have been released. The bulk of the changes are to runtime engine performance, and portability to MacOSX. See the NEWS file for other changes.

11 May 2002:

Version 0.3.6 of pnet and version 0.2.0 of pnetlib, have been released. The bulk of the changes are to the runtime engine and the C# compiler. Version 0.0.2 of pnetmark has also been released. See the PNetMark FAQ for more information.

20 April 2002:

Version 0.3.4 of pnet, version 0.1.8 of pnetlib, and version 0.1.0 of treecc have been released. Many changes to the runtime engine and pnetlib. The runtime engine has a new method caching policy and many bug fixes. Pnetlib has extensive support for internationalization (I18N) routines in separate plug-in assemblies. Some of the annoying automake 1.5 problems under Cygwin should now be fixed in the .tar.gz release versions (the CVS version may still be problematic with automake 1.5).

Version 0.3.2 of pnet, version 0.1.6 of pnetlib, and version 0.0.8 of treecc have been released. The biggest change is to the C# compiler, which can now compile the metadata in pnetlib (metadata consists of the class, field, and method definitions, but not the method bodies). The library, pnetlib, is also moving forward with many changes to the "System" assembly, including network code.
